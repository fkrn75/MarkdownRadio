# 마크다운 라디오 소개

마크다운 라디오는 **읽을 시간이 없을 때** 글을 *흘려들으며* 1차 필터링하고, 마음에 드는 대목을 북마크해 두었다가 나중에 정독하는 폐루프 도구다. 핵심은 음질이 아니라 `텍스트 정제`와 북마크→원문 점프의 정확성이다.

## 왜 만드는가

하루에 쌓이는 아티클·릴리스 노트·회의록은 많고 정독할 시간은 적다. 출퇴근길에 흘려듣다가 "이 부분 다시 보자" 싶은 지점만 표시해 두면, 책상 앞에서는 그 지점부터 정독하면 된다.

> 듣기는 1차 필터, 읽기는 2차 정독. 이 둘을 끊김 없이 잇는 것이 목표다.

### 지원 입력

- 마크다운 파일(`.md`, `.markdown`)
- 일반 텍스트(`.txt`)
- 클립보드 붙여넣기

자세한 배경은 [기획 문서](https://example.com/prd)를 참고하라. 아래 표는 처리 방식을 요약한다.

| 요소 | 청취(정제본) | 정독(원문) |
|------|--------------|------------|
| 코드 블록 | 건너뜀 | 그대로 표시 |
| 링크 | 텍스트만 | 링크 유지 |
| 표 | 건너뜀 | 그대로 표시 |

## 코드는 읽지 않는다

아래 같은 코드 블록은 청취에서 제외되어 소음을 막는다. 정독 화면에서는 원문 그대로 보인다.

```ts
function chunkify(text: string): Chunk[] {
  // 문장 단위로 쪼갠다
  return segment(text)
}
```

---

마지막으로, 모든 데이터는 기기 안에만 저장된다. 서버로 전송되는 정보는 없다. 그러니 안심하고 흘려들어도 된다.
