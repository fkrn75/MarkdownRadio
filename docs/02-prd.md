# 02 · 기획서 (PRD)

> Markdown Radio — AI 정보 소비용 성우 라디오 웹앱
> 버전 0.1 · 2026-06-21

---

## 1. 한 줄 정의

**한국어 마크다운/텍스트 자료를 브라우저 안에서 음성으로 바꿔, 라디오처럼 흘려듣고(1단계) 중요한 곳만 북마크해 나중에 정독(2단계)하는 무료 온디바이스 웹앱.**

---

## 2. 배경과 문제 정의

### 2.1 문제

AI 검색(Perplexity, Gemini 등), 논문, 테크 블로그에서 쏟아지는 **마크다운 기반 자료의 양이 과다**하다. 모든 자료를 눈으로 정독하기엔:
- 피로감이 크고 시간이 오래 걸린다.
- 어떤 자료가 정독할 가치가 있는지 **사전 필터링**이 어렵다.
- 이동 중(출퇴근·산책)에는 화면을 볼 수 없어 자료 소비가 멈춘다.

### 2.2 해결 가설

정보 습득을 **두 단계로 물리적으로 분리**한다:

| 단계 | 행동 | 목적 | 상황 |
|------|------|------|------|
| **1단계** | 귀로 흘려듣기 | 맥락 파악 · 정독할 자료 필터링 | 이동 중, 멀티태스킹 |
| **2단계** | 눈으로 정독 | 중요 내용 깊이 읽기 | 책상 앞, 집중 시간 |

두 단계를 잇는 다리가 **북마크**다. 1단계에서 "어, 이거 중요한데?" 싶을 때 버튼을 누르면 그 지점이 기록되고, 2단계에서 북마크만 열어 해당 문맥을 바로 정독한다.

### 2.3 왜 지금, 왜 이렇게

- 2026년 기준 **브라우저 안에서 100% 로컬로 도는 한국어 TTS 모델**(Supertonic)이 실용 수준에 도달 → 서버·비용 없이 무료로 가능.
- 기존 TTS 앱/서비스는 (a)클라우드 비용·프라이버시 문제, (b)마크다운 자료 특화 부재, (c)"흘려듣기→북마크→정독" 워크플로 부재.

---

## 3. 타겟 사용자

### 3.1 주 사용자 (페르소나)

**"정보 과식자" — 개발자/연구자/지식 노동자**
- AI 검색·논문·기술 문서를 하루에도 수십 건 마주친다.
- 출퇴근/산책 시간을 정보 소비에 쓰고 싶어 한다 (에어팟 상시 착용).
- 한국어 자료가 중심이지만 코드·영문 용어가 섞여 있다.
- 프라이버시·비용에 민감해 "내 자료가 서버로 안 가는" 온디바이스를 선호한다.

### 3.2 사용 환경

- **PC**: 책상에서 자료 정리하며 청취 + 정독 (1급).
- **모바일**: 이동 중 청취 (1급). Android가 백그라운드 재생에 유리, iOS는 제약 있음(§8).

---

## 4. 핵심 가치 제안

1. **공짜이고, 영원히 공짜다.** 사용자 기기가 연산하므로 서버·API 비용이 구조적으로 0. 무료 티어 종료 리스크 없음.
2. **내 자료가 내 기기를 안 떠난다.** 온디바이스 추론 → 완전한 프라이버시. 오프라인 동작.
3. **마크다운을 안다.** `#`, `*`, 코드블록, 표를 기계적으로 읽지 않고 자연스럽게 정제·연출한다.
4. **듣기와 읽기를 잇는다.** 북마크가 흘려듣기(1단계)와 정독(2단계)을 매끄럽게 연결한다.

---

## 5. 핵심 사용자 시나리오

### 시나리오 A — 출근길 흘려듣기
1. 전날 밤 PC에서 AI 검색 결과 5건을 `.md`로 저장 → Markdown Radio에 업로드해 "재생 목록"에 넣어둔다.
2. 아침 지하철에서 폰으로 홈 화면 아이콘을 켜고, 에어팟으로 재생. 1.25배속.
3. 3번째 자료에서 흥미로운 대목 → **북마크 버튼 탭**. 계속 듣는다.
4. 사무실 도착. 북마크 2개가 기록돼 있다.

### 시나리오 B — 책상에서 정독
1. PC에서 북마크 목록을 연다.
2. 북마크를 클릭 → 원문에서 해당 문장으로 **자동 스크롤 + 하이라이트**.
3. 앞뒤 문맥을 눈으로 정독. 필요하면 그 지점부터 다시 재생.

### 시나리오 C — 오프라인
1. 비행기 모드. 이미 1회 다운로드된 TTS 모델이 IndexedDB에 있으므로 그대로 동작.
2. 저장해 둔 문서를 청취. (새 모델 다운로드만 불가)

## 7. 범위 (Scope)

### 7.1 MVP (v1.0) — In Scope

- 문서 입력: 파일 업로드(`.md`, `.txt`) + 텍스트 직접 붙여넣기
- 마크다운 정제 (헤더/강조/링크/코드블록/표/리스트 처리)
- 문장 청크 분할 + 원문 위치 메타
- TTS 합성 (Supertonic, transformers.js) + 모델 IndexedDB 캐싱·진행률
- 연속 재생 (재생/일시정지/이전·다음 문장/seek)
- 배속 (0.75 ~ 2.0x, 피치 보존)
- 북마크 (추가/삭제/목록) + 정독 뷰(원문 하이라이트·점프)
- 문서 라이브러리 (저장/삭제, IndexedDB)
- PWA (홈 화면 추가, 오프라인, MediaSession 잠금화면 컨트롤)
- PC + Android + iOS(제약 명시) 동작

### 7.2 Post-MVP (v1.x+) — 향후

- 모델 선택 UI (Supertonic / MeloTTS / 화자·속도 프리셋)
- 제목 전환 시 의도적 무음(라디오 코너 연출)
- 재생 목록(여러 문서 연속 재생) · 자동 진행
- URL 가져오기(웹페이지 → 본문 추출 → 읽기)
- 클라우드 동기화(선택적, 북마크/문서) — 단 온디바이스 원칙 유지
- 다국어(영어 자료 별도 모델 분기)
- 북마크에 메모/하이라이트 색상

### 7.3 Out of Scope (명시적 제외)

- 서버사이드 TTS·계정·로그인 (온디바이스 원칙 위배)
- 음성 클로닝·커스텀 보이스 학습
- 실시간 협업·공유 서버
- 네이티브 앱(iOS/Android 스토어) — 웹/PWA로 한정

## 8. 비기능 요구사항 (NFR)

| 항목 | 목표 |
|------|------|
| 비용 | 운영비 0원 (서버·API 없음) |
| 프라이버시 | 문서·음성이 사용자 기기를 벗어나지 않음 |
| 오프라인 | 모델 1회 다운로드 후 완전 오프라인 동작 |
| 첫 로딩 | 모델 다운로드 진행률 표시, 2회차부터 캐시로 즉시 |
| 합성 성능 | 재생이 합성을 추월하지 않도록 다음 청크 선행 합성 (체감 끊김 없음) |
| 기기 호환 | PC(Chrome/Edge/Firefox/Safari) + 모바일. WebGPU 없으면 WASM 폴백 |
| 접근성 | 키보드 컨트롤, 적절한 폰트 크기, 다크모드 |

## 10. 성공 지표 (개인 프로젝트 기준)

- **정성**: "출퇴근길에 실제로 매일 쓰게 되는가?" (본인 도그푸딩)
- 정독 전환율: 흘려듣기 중 북마크 → 실제 정독으로 이어지는 비율
- 첫 문장 재생까지 시간(2회차, 캐시 상태) < 3초
- 1시간 청취 중 체감 끊김 0회 (PC/Android)

## 11. 마일스톤

| 단계 | 산출물 | 검증 |
|------|--------|------|
| **M0** 검증 | Supertonic 데모 실청취 + 최소 PoC | 한국어 음질 합격 판정 |
| **M1** 텍스트 파이프라인 | 업로드→정제→청크→원문메타 | 샘플 MD가 깔끔한 평문 청크로 |
| **M2** 합성·재생 | engine 추상화 + 1문장 합성·재생 | 한 문장이 소리난다 |
| **M3** 연속 재생 | 워커+프리페치+배속+모델 캐싱 | 긴 문서 끊김 없이 재생·배속 |
| **M4** 북마크·정독 | 북마크 저장/목록 + 정독 뷰 점프 | 북마크→원문 하이라이트 |
| **M5** PWA·모바일 | SW+manifest+MediaSession+iOS 경로 | 홈 화면 추가, 잠금화면 컨트롤 |
| **M6** 배포 | Cloudflare Pages 배포 | 폰에서 URL 접속·설치 |

상세 태스크는 [04-checklist.md](04-checklist.md).
